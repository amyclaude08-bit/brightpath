# BrightPath AI Operations System

A lightweight multi-agent AI assistant for a small training company. Handles inbound sales leads, customer service requests, and produces manager briefings — powered by Claude.

---

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Set your Anthropic API key

```bash
export ANTHROPIC_API_KEY=your_key_here
```

Or add it to a `.env` file and load it (see below).

### 3. Run

```bash
python orchestrator.py
```

## How It Works

Every inbound message goes through three steps:

1. **Triage Agent** — classifies the message, detects urgency and sentiment, identifies the customer, routes to the right specialist
2. **Specialist Agent** — Sales or Customer Service — reads memory files, applies policies, drafts a response, recommends next action
3. **Manager Briefing Agent** — produces a short Telegram-ready summary of what happened and what needs human attention

All events are logged to SQLite. Tasks generated by agents are stored and viewable.

## Adding Telegram

To add Telegram as an input/output channel:

```bash
pip install python-telegram-bot
```

Create `telegram_bot.py` — receive messages and pass them to `process_message()` from `orchestrator.py`. Return the `telegram_summary` from the briefing agent back to the chat.

This is a 50-100 line addition and doesn't change any agent logic.

---

## Customising Agents

All agent behaviour is defined in the `.md` files in `agents/`. Edit these to change:
- Roles and responsibilities
- Pricing
- Policies (refunds, cancellations)
- Tone guidelines
- Output rules

No Python changes needed for behaviour changes.

---

## Memory Updates

Update the files in `memory/` to keep context fresh:
- Add new customers to `customer_history.md`
- Update `sales_notes.md` as deals progress
- Clear completed tasks from `open_tasks.md`

Future versions can have agents write back to these files automatically.
